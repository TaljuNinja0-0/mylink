# 마이링크 (MyLink) 기능 정의서 (PRD)

## 1. 프로젝트 개요

### 1-1. 프로젝트명
**마이링크 (MyLink)**

### 1-2. 목적 (Objective)
개발자 및 크리에이터가 자신들의 흩어진 작업물(GitHub 저장소, 블로그, 포트폴리오, 소셜 채널)을 하나의 링크(`mylink.app/[URL슬러그]`)로 가볍게 통합하여 제공할 수 있는 심플한 멀티링크 랜딩 페이지 서비스.

### 1-3. 타겟 사용자 (Target Audience)
- **개발자 (Developers)**: GitHub, 기술 블로그, 이력서 등을 직관적으로 모아 취업, 네트워킹에 활용하고자 하는 IT 종사자
- **크리에이터 (Creators)**: 유튜브, 인스타그램 등의 채널 트래픽을 한곳에 효과적으로 모아 포트폴리오화 하려는 1인 미디어

### 1-4. 주요 디자인 및 차별화 (USP)
- **모던 디자인 (shadcn/ui)**: 접근성 높고 세미-엔터프라이즈 느낌의 세련된 shadcn/ui 기반의 미니멀리즘 프로필 레이아웃 기본 탑재
- **직관적인 인라인 편집 (Inline Editing)**: 텍스트를 마우스로 클릭하여 그 자리에서 즉시 내용을 수정할 수 있는 가장 직관적인 편집 UX 환경 제공.
- **직관적인 통계 제공**: 내 프로필 전체 방문자 수와 링크 클릭 성과를 한눈에 확인할 수 있는 통계 시각화

## 2. 핵심 기능 목록 (전체 필수 구현)

*참고: 본 서비스는 유저 허들을 극복하기 위해 오직 기능성에 집중하며 이미지 업로드, 선택 부가 기능(Nice-to-have) 등을 모두 배제하여 필수 기능(MVP)만을 가볍게 지원합니다.*

### 2-1. 회원 인증 시스템 (Auth)
- **단일 소셜 로그인 (Google Only)**
- **Firebase Authentication**을 이용한 이메일 폼 없는 간편 가입 및 로그인 프로세스
- **지메일 기반 URL 슬러그(`displayName`) 자동 할당 체계**: 최초 구글 로그인 가입 시, 사용자의 **구글 지메일(Gmail) 아이디 앞부분(`@` 기호 이전)**을 파싱하여 초기 URL 슬러그(`displayName`) 값으로 즉시 할당함. 이를 통해 `mylink.app/[지메일앞부분]` 형태의 사용자 고유 주소가 생성되어 가입 허들을 획기적으로 낮춤.
- **데이터베이스 (DB)**: **Firebase Firestore (NoSQL)** 채택 확정. 유저 정보 중심의 서브 컬렉션(Subcollection) 모델링 적용.

### 2-2. 프로필 관리 기능 (Profile Management) - 인라인 편집 전용
- **텍스트 기반 프로필 구성**: 아바타 및 프로필 사진 업로드 기능을 전면 지양하며 오직 텍스트 직관성 위주의 미니멀리즘 뷰어 구축
- **기본 정보 인라인 수정 (Inline Editing)**: 사용자는 설정 화면으로 이동할 필요 없이, 대시보드 화면상에 표시되는 **1) 접속 슬러그(`displayName`)**, **2) 실제 노출 이름(`username`)**, **3) 자기소개(`bio`)** 항목의 텍스트를 마우스로 바로 클릭해 즉시 타이핑하여 실시간 수정/보존을 처리함.

### 2-3. 링크 관리 대시보드 (Link Management)
- **생성, 수정, 삭제 (CRUD)**: 직관적인 구조로 버튼 타이틀(Title)과 대상 URL 주소를 생성/삭제 처리함.
- **링크정보 인라인 수정**: 기존에 만들어둔 링크 역시 상세 화면 이동 없이 마우스로 텍스트를 클릭하여 그 자리에서 텍스트(타이틀 및 링크)를 바로 덮어쓰고 확정(Inline Editing)함.
- **파비콘(Favicon) 자동 연동 (Google API)**: 별도의 수동 업로드나 무거운 스크래퍼 대신, 빠르고 가벼운 `Google Favicon API`를 호출하여 입력된 URL 도메인의 공식 식별 아이콘을 즉시 추출해 좌측에 자동 매핑함.
- *(제한 사항: 링크 위치 이동 재정렬, 숨김 및 알림 토글, 그리고 **테마/색상 커스터마이징(Theme)** 등의 확장 기능은 오버엔지니어링 방지 및 미니멀리즘 디자인 철학 유지를 위해 전면 금지(단일 테마 강제 탑재)됨.)*

### 2-4. 퍼블릭 화면 렌더링 (Public Profile)
- 사용자 고유 URL 슬러그 기반 접속 시 모바일 비율에 맞춰 세로 배열된 `shadcn/ui` 프레임워크 뷰로 일반 방문자에게 포트폴리오 랜딩 페이지를 즉각 렌더링함.

### 2-5. 방문자 통계 분석 (Analytics Dashboard)
- 프로필 페이지 단위의 전체 누적/일일 **방문자 수 현황** 조회 기능 지원
- 개별 등록된 각 링크 항목별로 **클릭 조회수(Click Counts)** 실시간 트래킹 및 숫자 카운터 표출
