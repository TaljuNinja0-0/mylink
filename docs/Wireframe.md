# MyLink 와이어프레임 (Wireframe)

이 문서는 MyLink 프로젝트의 3가지 핵심 화면(메인 홈페이지, 관리자 대시보드, 퍼블릭 랜딩 페이지)에 대해 시각적인 레이아웃 구조와 내비게이션 흐름을 마크다운, 다이어그램, 텍스트 아트 서식을 혼합하여 정의합니다.

---

## 1. 페이지 간 탐색 흐름 (Mermaid Flowchart)

먼저 사용자가 서비스를 오가는 논리적인 화면 전환의 흐름을 보여줍니다.

```mermaid
graph TD
    classDef page fill:#f9f,stroke:#333,stroke-width:2px;
    classDef action fill:#bbf,stroke:#333,stroke-width:2px;
    classDef external fill:#bfb,stroke:#333,stroke-width:2px;

    Home[메인 홈페이지<br/>(로그인 전)]:::page -->|Google Auth 연동 클릭| Login[로그인 및 계정 생성]:::action
    Login -->|동기화 및 세션 할당| Dash[관리자 대시보드<br/>/dashboard]:::page
    
    Dash -->|자기소개/이름 텍스트 클릭| Inline1[인라인 편집 진행]:::action
    Dash -->|새로운 링크 추가 폼 전송| Links[(링크 데이터 추가)]
    Dash -->|링크 제목/URL 클릭| Inline2[인라인 편집 진행]:::action
    
    Dash -->|상단에 안내된 <br/>본인의 URL 경로 클릭| Public[퍼블릭 랜딩 페이지<br/>/displayName]:::page
    Public -->|화면에 나열된<br/>링크 블록 클릭| Ext[외부 목적지 사이트로 이동<br/>target=\"_blank\"]:::external
    Public -.->|Firestore 백그라운드| ClickCount[(클릭 조회수 +1 업데이트)]
```

## 2. 퍼블릭 랜딩 페이지 (Public View) - ASCII Art

방문자가 `mylink.com/displayName` 경로로 접속했을 때 보이는 최종 결과물입니다. 불필요한 장식을 배제한 심플하고 텍스트 지향적인 디자인을 지향합니다.

```text
+-------------------------------------------------------------+
|                                                             |
|                          [ 사진 ]                           |
|                        (Google img)                         |
|                                                             |
|                   username (실제 이름)                      |
|                     user@gmail.com                          |
|                                                             |
|         "저의 마이링크 포트폴리오를 방문해주셔서              |
|          감사합니다. 아래 링크를 확인해 보세요!"               |
|                                                             |
|    +---------------------------------------------------+    |
|    | [Favicon]     나의 첫번째 외부 웹사이트           |    |
|    +---------------------------------------------------+    |
|                                                             |
|    +---------------------------------------------------+    |
|    | [Favicon]     GitHub 레포지토리                   |    |
|    +---------------------------------------------------+    |
|                                                             |
|    +---------------------------------------------------+    |
|    | [Favicon]     개인 블로그 주소 (Velog)            |    |
|    +---------------------------------------------------+    |
|                                                             |
|                         [ MyLink ]                          |
+-------------------------------------------------------------+
```

## 4. UI 컴포넌트 구성 방식

- **파비콘(Favicon) 이미지**: 타사 목적지 URL을 파싱한 뒤 `https://www.google.com/s2/favicons?domain={URL}` 엔드포인트를 호출하여 이미지를 즉시 렌더링합니다.
- **인라인 에디터(Inline Editor)**: 일반적인 `<p>` 또는 `<span>` 텍스트 레이블로 보여지고, `onClick` 이벤트가 발생하면 `shadcn`의 `<Input>` 인풋 필드로 부드럽게 전환(Toggle)되도록 구현합니다. 블러(Blur)나 엔터(Enter) 이벤트 발생 시 Firestore에 변경 사항을 자동 저장(Auto-save)합니다.
- **레이아웃(Layout)**: TailwindCSS의 Flexbox 클래스를 적극 활용하여, 세로로 긴 링크 트리를 중앙에 깔끔하게 정렬(`flex-col items-center max-w-md mx-auto`)하는 모바일 중심(Mobile-first) 형태로 설계합니다.
