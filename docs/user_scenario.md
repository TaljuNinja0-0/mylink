# 마이링크 (MyLink) 도메인 사용자 시나리오 (User Scenario)

아래 시나리오는 핵심 대상인 **방문자(Visitor)** 와 **소유자(Owner)** 의 관점에서 예측할 수 있는 모든 행동 양상을 "사용자는 [목적]하기 위해 [행동]한다." 형식으로 포괄하여 서술한 문서입니다.

---

## 1. 방문자 (Visitor): 랜딩 페이지 접속 및 탐색 과정

1. 방문자는 **소유자가 연결해 놓은 모든 웹 소스를 한눈에 파악하기 위해** 외부 채널이나 명함 등에 기재된 소유자의 개인 전용 URL(`mylink.com/displayName`) 브라우저에 입력하거나 클릭하여 접속한다.
2. 방문자는 **도착한 이 페이지의 진짜 주인이 자신이 찾던 사람이 맞는지 직관적으로 인지하기 위해** 화면 최상단에 있는 구글 연동 프로필 이미지와 실제 사용자 이름(`username`)을 확인한다.
3. 방문자는 **소유자에게 직접 비즈니스 문의나 연락을 취하기 위해** 상단 프로필 영역에 노출된 이메일 주소를 읽거나 복사한다.
4. 방문자는 **소유자의 현재 상태나 서비스의 목적을 파악하기 위해** 짧은 자기소개(Bio) 텍스트를 읽어본다.
5. 방문자는 **각 링크가 어떤 사이트(예: GitHub, YouTube, Velog 등)로 연결되는지 구별하기 위해** 제공된 개별 링크 좌측의 파비콘(Favicon) 아이콘을 시각적으로 훑어본다.
6. 방문자는 **소유자가 등록한 특정 링크의 상세 내용을 텍스트로 읽기 위해** 파비콘 옆에 쓰여 있는 링크 제목(Title)을 확인한다.
7. 방문자는 **관심 있는 소유자의 특정 웹사이트 혹은 깃허브 저장소 등으로 이동하기 위해** 원하는 링크 블록 영역을 클릭(모바일의 경우 탭)한다.
8. *(시스템 동작)* 방문자가 링크를 클릭하는 순간 클릭 통계 집계가 작동하여 Firestore의 클릭 조회수(clickCount)가 증가하고, 방문자는 브라우저의 새 창(`target="_blank"`)에서 목적지 사이트 화면을 쾌적하게 보게 된다.
9. 방문자는 **모바일, 태블릿, 데스크톱 등 다양한 디스플레이 기기 환경에서 깨짐 없는 화면을 보기 위해** 반응형 웹으로 렌더링 된 마이링크 페이지를 위아래로 스크롤하며 자유롭게 탐색한다.

---

## 2. 소유자 (Owner): 계정 생성 및 대시보드 관리(CRUD) 과정

1. 소유자는 **마이링크 서비스를 빠르고 간편하게 이용하기 위해** 메인 페이지에서 'Google 소셜 로그인' 버튼을 선택하여 별도의 폼 작성 없이 가입 및 로그인을 완료한다.
2. 소유자는 **자신의 구글 지메일 아이디 앞부분이 파싱되어 개인 URL(`displayName` 식별자)이 잘 생성되었는지 확인하기 위해** 로그인 직후 자동으로 이동된 관리자 대시보드 상단 영역을 살핀다.
3. 소유자는 **자신의 구글 프로필 이미지와 이메일 데이터가 제대로 동기화되었는지 확인하기 위해** 대시보드 및 퍼블릭 뷰에 고정으로 표시된 사진과 이메일을 확인한다.
4. 소유자는 **초기에 설정된 사용자 이름(`username`)을 본인의 진짜 이름이나 활동명으로 변경하기 위해** 이름 텍스트 영역을 마우스로 클릭하여 대화형 입력창(인라인 편집)을 띄운 뒤 즉시 내용을 타이핑하고 확정한다.
5. 소유자는 **방문자들에게 방문 환영 인사나 자신의 포트폴리오를 요약해서 전달하기 위해** 자기소개(Bio) 영역을 클릭하여 인라인 편집 방식으로 텍스트를 입력하고 저장한다.
6. 소유자는 **방문자들에게 제공할 새로운 외부 웹사이트나 자신의 콘텐츠를 공유하기 위해** 대시보드에 있는 '새 링크 추가' 버튼을 눌러 제목과 목적지 URL 폼을 호출한다.
7. 소유자는 **방문자가 정확한 장소로 갈 수 있도록 유도하기 위해** 호출된 폼 안에 링크의 제목과 목적지 주소(https://...)를 정확히 입력한 뒤 추가(저장) 버튼을 누른다.
8. 소유자는 **새로 등록한 링크 항목에 타겟 사이트의 로고가 잘 나오는지 확인하기 위해** Google Favicon API가 호출되어 링크 좌측에 파비콘 이미지가 생성된 것을 눈으로 점검한다.
9. 소유자는 **이미 등록한 링크의 제목이나 URL 텍스트 주소에 오타가 생겨 수정하기 위해** 해당 문구를 직접 클릭하여 인라인 편집(Inline Editing) 방식으로 빠르게 에러 문구를 정정한다.
10. 소유자는 **기간이 끝난 이벤트나 더 이상 유효하지 않은 포트폴리오 사이트를 리스트 뷰에서 지우기 위해** 해당 링크 항목 우측에 위치한 삭제 아이콘(휴지통 등)을 클릭하여 확인 절차 후 완전히 제거한다.
11. 소유자는 **자신의 어떤 링크가 사람들에게 가장 인기가 있었는지 성향을 분석하기 위해** 대시보드를 방문하여 각 링크 하단 혹은 우측에 표시된 누적 '클릭 조회수' 숫자를 주기적으로 점검한다.
12. 소유자는 **자신이 편집한 마이링크 대시보드 설정이 대중들에게 실제로 어떻게 렌더링되는지 최종 점검하기 위해** 화면에 표시된 자신의 퍼블릭 링크 주소(`mylink.com/displayName`)를 클릭하여 방문자 입장에서 랜딩 페이지를 시뮬레이션한다.
13. 소유자는 **개인 정보 보안을 지키고 자신의 작업을 안전하게 마무리하기 위해** 대시보드 메뉴에 있는 '로그아웃' 버튼을 클릭해 현재 기기에서의 세션을 완전히 종료시킨다.
