# 마이링크 (MyLink) 사용자 시나리오 (User Scenario)

## 1. 방문자 (Visitor): 마이링크 페이지 방문 과정

- 방문자는 **특정 크리에이터의 전체 작업물과 소셜 링크들을 한눈에 확인하기 위해** 인스타그램, 유튜브 등에 공유된 `mylink.app/[displayName]` 형태의 **단일 URL을 클릭하여 접속한다.**
- 방문자는 **서비스에 가입 및 로그인할 필요 없이 즉시 내용을 열람하기 위해** 접속과 동시에 인증 로직 없이 렌더링된 **단일 페이지 화면을 위아래로 스크롤하며 탐색한다.**
- 방문자는 **자신이 원하는 목적지(예: 깃허브, 개별 블로그 포스팅 등)로 이동하기 위해** 화면에 나열된 파비콘이 포함된 **링크 버튼을 직접 클릭한다.**

## 2. 소유자 (Owner): 링크 추가/수정/삭제 과정

### 링크 추가 (Create)
- 소유자는 **새로운 바로가기 링크 버튼을 화면에 무에서 유로 만들기 위해** 프로필 페이지 하단에 위치한 **'+ 새 링크 추가' 버튼을 빈 템플릿 형태로 클릭한다.**
- 소유자는 **생성된 빈 버튼에 안내할 내용과 실제 이동할 주소를 연결하기 위해** 해당 버튼의 텍스트 영역을 직접 클릭하고 **제목과 URL을 타이핑한 뒤 엔터(Enter) 키(또는 바깥 영역 클릭)를 눌러 저장한다.** (인라인 편집)
- 소유자는 **웹사이트의 아이콘을 버튼 좌측에 달기 위해** 별도의 이미지를 업로드하지 않고, URL이 수정 저장됨과 동시에 시스템이 **자동으로 구글 파비콘 API를 호출해 그려주기를 기다린다.**

### 링크 수정 (Update)
- 소유자는 **과거에 등록해 둔 링크의 오타를 고치거나 주소를 변경하기 위해** 별도의 복잡한 '설정(관리자) 창'으로 이동하지 않고, 화면상에 렌더링된 버튼의 텍스트를 **마우스로 바로 클릭한 후 인라인 폼으로 변환되면 즉시 타이핑하여 덮어쓴다.**

### 링크 삭제 (Delete)
- 소유자는 **더 이상 유효하지 않거나 홍보하고 싶지 않은 링크를 제거하기 위해** 해당 버튼 우측 가장자리에 위치한(혹은 호버 시 나타나는) 소형 **'휴지통(삭제)' 아이콘 컴포넌트를 클릭하여 아이템을 영구 삭제한다.**
